Jupter Notebook Structure#

Jupter Notebooks are build out of cells. If you double click on the text, you’ll see that these words are written in a cell that can be edited and moved around and deleted. Cells have different types:

  • Markdown means you’re writting in the cell, not coding.

  • code means you’re coding inside the cell. We can use a combination of markdown cells and coding cells to help keep track of what’s happening in the notebook.

There are two modes for cells: command (which is running the cell) and edit (self explainitory, the cell is not being run but instead is being edited.)

  • to enter command mode, click outside the cell. This will stop any typing/coding from continuing in the cell and allow you to run the cell

  • to enter edit mode, double click the cell. Alternativly, you can click the cell then hit enter or return.

shortcuts:

  • Ctrl + Enter (mac is Command + Enter) run selected cells

  • Shift + Enter run the current cell, select below

  • Alt + Enter (mac is Option + Enter) run the current cell, insert below

  • D + D (press the key twice) to delete the current cell

  • if you’re in command mode (click outside the cell), press ‘a’ to add cell above and ‘b’ to add cell below

Markdown vs Code#

You can select the desired format from a dropdown menu at the top. If you want to type words, use “Markdown”; if you want to write code, choose “Code”. Or you can use the shortcuts:

  • M to change the current cell to Markdown cell, Y to change it to code cell.

Conditionals#

Conditionals are very common in codeing as it allows us to automate alot of processes. They usually look like this:

if (test):

If test is true, run

elif (second test):

if the first test is false but second test is true, run

else:

  if both tests are false, run this part

x = 20
if x>10:
    print('wow this is greater than 20')
elif x == 10:
    print('this number is 10')
else:
    print('sad this number is smaller than 10')

wow this is greater than 20
